# 隐私说明

NoticeFlow 是一个本地优先的 macOS 应用。本文说明 NoticeFlow 会读取什么数据、在本机保存什么数据，以及哪些情况下数据可能离开本机。

## NoticeFlow 会读取的数据

NoticeFlow 可能会读取：

- 本机 `usernoted` 数据库中的 macOS 通知记录。
- 通知字段，例如应用标识、应用名称、标题、副标题、正文、通知时间、线程标识，以及通知文本中识别到的 URL。
- 构建应用选择器时需要的本机应用元数据和图标。
- 用户选择的脚本、规则文件，以及配置在 NoticeFlow 数据目录中的设置。

读取 macOS 通知数据库需要本机权限，例如完全磁盘访问权限。

## NoticeFlow 会保存的数据

NoticeFlow 会把以下数据保存在本机：

- 规则和设置。
- 名为 `notifications.sqlite` 的本地通知归档。
- 隐藏通知状态和动作执行历史。
- 本地脚本执行时产生的临时输出日志。

具体保存位置取决于你配置的 NoticeFlow 数据目录。

## NoticeFlow 会发送的数据

NoticeFlow 默认不会把通知数据发送到远程服务。

但用户配置的自动化动作可能会把数据发送到本机之外，包括：

- HTTP 请求动作。
- 规则中配置的 webhook。
- 调用网络工具的 Shell、Python、JavaScript 或 AppleScript 动作。
- 把通知变量拼接进浏览器 URL 的打开链接动作。

启用规则前请先检查规则内容，尤其是会执行本地脚本或发送 HTTP 请求的规则。

## 删除系统通知

NoticeFlow 只能在用户显式操作时删除选中的系统通知记录。该操作会写入本机 macOS 通知数据库，并且可能无法恢复。

## 备份与诊断信息

不要公开包含通知内容的本地数据库备份、动作日志、崩溃日志或截图。仓库默认会忽略本地 `backups/` 数据。
