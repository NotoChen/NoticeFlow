# 安全说明

## 支持版本

NoticeFlow 目前仍是 1.0 前的软件。安全修复默认面向主开发分支的最新版本，除非项目后续明确维护独立 release 分支。

## 报告安全问题

请先私下报告安全问题，不要直接公开提交 issue。

如果仓库暂未启用 GitHub private vulnerability reporting，可以通过 `NotoChen` 的 GitHub 主页联系维护者。

报告时请尽量包含：

- 受影响的版本或 commit。
- macOS 版本。
- 复现步骤。
- 影响范围和预期行为。
- 已移除个人通知内容的日志或截图。

## 安全模型

NoticeFlow 是本地自动化工具，最重要的边界是用户意图。

- 规则是用户控制的本地自动化配置。
- 脚本动作会以当前用户权限执行本地命令。
- HTTP 动作和脚本可能会在用户配置后发送通知内容。
- 完全磁盘访问权限会允许 NoticeFlow 读取本机 macOS 通知数据。

不要启用来自不可信来源的规则。运行前请检查脚本动作。

## 不视为漏洞的预期行为

以下行为在用户明确配置后属于预期行为：

- 规则执行本地脚本。
- 规则通过 HTTP 动作发送通知内容。
- 规则打开包含通知变量的 URL。
- 用户显式删除本机 macOS 通知数据库中的通知记录。
